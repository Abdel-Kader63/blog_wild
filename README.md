En ce qui concerne ce projet, j'ai dû faire un blog sur le thème de mon choix. Le projet doit inclure une page de présentation, une page qui répertorie les articles disponibles, ainsi qu'une page pour afficher le contenu d'un article spécifique. En termes de fonctionnalités, un utilisateur doit pouvoir s'inscrire, se connecter et une fois connecté doit pouvoir créer, modifier et supprimer ses articles sur son espace personnel. N'importe qui peut consulter les articles des autres utilisateurs et a la possibilité de consulter un article en détail. Un utilisateur peut laisser un commentaire en dessous d’un article (à condition d’être connecté). J’ai par conséquent dû créer une plateforme de blog en PHP/Laravel avec une gestion des utilisateur·ices.

Vous pouvez récupérer mon projet en effectuant la commande suivante : git clone git@github.com:nom-utilisateur/nom-depot.git

Naviguez dans le répertoire du projet cloné en utilisant la commande cd : cd nom_du_projet

Créez un fichier .env en copiant le contenu du fichier .env.example : cp .env.example .env

Générez une nouvelle clé d'application Laravel en utilisant la commande : php artisan key:generate

Installez les dépendances du projet en utilisant la commande : composer install

Une fois l'installation terminée, vous pouvez lancer l'application en utilisant la commande : php artisan serve

Assurez-vous d'avoir PHP, Composer et Laravel (via Composer) installés sur votre machine avant de cloner et exécuter un projet Laravel.

## About Laravel

Laravel is a web application framework with expressive, elegant syntax. We believe development must be an enjoyable and creative experience to be truly fulfilling. Laravel takes the pain out of development by easing common tasks used in many web projects, such as:

- [Simple, fast routing engine](https://laravel.com/docs/routing).
- [Powerful dependency injection container](https://laravel.com/docs/container).
- Multiple back-ends for [session](https://laravel.com/docs/session) and [cache](https://laravel.com/docs/cache) storage.
- Expressive, intuitive [database ORM](https://laravel.com/docs/eloquent).
- Database agnostic [schema migrations](https://laravel.com/docs/migrations).
- [Robust background job processing](https://laravel.com/docs/queues).
- [Real-time event broadcasting](https://laravel.com/docs/broadcasting).

Laravel is accessible, powerful, and provides tools required for large, robust applications.
